## 0. 设计审计结论

### 当前问题（为何「丑」）

| 问题 | 表现 | 原则违背 |
|------|------|----------|
| 无视觉层级 | 标题、表格、按钮同等权重 | 层次即服务 |
| 通用 SaaS 蓝 | `#2563eb` + 白卡片 = AI 模板感 | AI slop 风险 |
| 原生控件裸奔 | `<input type="file">`、`<select>` 未设计 | 像素级信任 |
| 无状态设计 | 无 loading / empty / 扫描中 / 失败 | 交互状态覆盖 |
| 信息密度失衡 | Step2 表格 + 手动补充堆在一起 | 信息架构 |
| 词库页半成品 | 「最近加入」仅 hint 文案 | 空状态不是设计 |
| 无任务上下文 | 左侧无历史、无进度、无文件名常驻 | 用户旅程 |

### 10 分长什么样（本工具）

审校师打开页面 **3 秒内** 知道：上传在哪、上次的任务在哪、当前进行到哪一步；扫描结果 **一眼区分** 词库命中 vs 手动补充；验证未通过时 **红色摘要 + 可展开残留列表**，而不是一行字。

## 1. 设计定位

### 1.1 产品人格：**审校工作台（Audit Workbench）**

- **不是** ClawSocial 暖色龙虾风（`/deid` 继续与主站分离）
- **不是** 通用 SaaS Dashboard（无 widget 网格、无 hero）
- **是** 专业、克制、文档中心的工具 — 类似「Adobe Acrobat 审阅模式 × 现代 Notion 表格」的交叉：可信、清晰、少装饰

### 1.2 情感目标

| 时刻 | 用户感受 | 设计支撑 |
|------|----------|----------|
| 首次进入 | 「这是正经工具，数据在本地」 | 深色顶栏 + 本地处理徽章 |
| 上传后 | 「系统在帮我找敏感信息」 | 扫描动画 + 命中数实时增加 |
| 确认实体 | 「我掌控哪些要脱」 | 表格可勾选、手动补充醒目 |
| 导出前 | 「我知道干不干净」 | 验证卡片：通过绿 / 未通过红 + 明细 |
| 词库维护 | 「越用越省事」 | 命中次数、最近沉淀可见 |

### 2.2 字体

```css
--deid-font: "IBM Plex Sans", "Source Han Sans SC", "Microsoft YaHei", system-ui, sans-serif;
--deid-font-mono: "IBM Plex Mono", "Consolas", monospace; /* 占位符、regex */
```

Google Fonts 引入：`IBM Plex Sans` + `IBM Plex Mono`（400/500/600）。

### 2.3 间距与圆角

- 间距基准：**4px** 网格（8 / 12 / 16 / 24 / 32）
- 圆角：`--deid-radius-sm: 6px`，`--deid-radius: 10px`，`--deid-radius-lg: 14px`
- 阴影：仅 elevated 面板 `0 1px 3px rgba(15,23,42,0.08), 0 4px 12px rgba(15,23,42,0.04)`

### 2.4 触控与 a11y

- 最小点击区域：**44×44px**（按钮、checkbox 行）
- 焦点环：`outline: 2px solid var(--deid-accent); outline-offset: 2px`
- 对比度：正文 ≥ 4.5:1；badge 文字 ≥ 4.5:1
- 表格：`scope="col"`、`aria-sort`；步骤条 `aria-current="step"`

## 4. 组件规格

### 4.1 顶栏 `DeidTopBar.vue`

- 左：产品名 + 副标「Word 主体脱敏 · 数据不出服务器」
- 右：Tab 切换（下划线式，非按钮组）
- 可选：「使用说明」折叠链接

### 4.2 任务侧栏 `DeidJobSidebar.vue`

| 元素 | 规格 |
|------|------|
| 任务卡片 | 文件名 truncate、状态 badge、剩余清理时间 countdown |
| 选中态 | 左边 3px primary 竖条 + surface 背景 |
| 空状态 | 插画线框 +「上传第一份底稿开始脱敏」+ 主按钮 |
| 新建 | 底部固定「+ 新建任务」 |

状态 badge：`draft` 灰 / `scanned` 蓝 / `done` 绿 / `failed` 红

### 4.3 步骤条 `DeidStepper.vue`

- 水平步骤：圆点 + 连线；完成=实心+勾；当前=accent 环；未到=灰
- 可点击已完成步骤回看（只读）

### 4.4 Step1 — 上传 `DeidStepUpload.vue`

**拖拽区**（非裸 file input）：

```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
│     📄  拖放 .docx 到此处        │
│        或 [ 选择文件 ]            │
│     单文件 · 最大 20MB           │
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

- 词库包：**chip 多选**（国家电投 / 中国能建 / 通用财务），默认全选 default packs
- 主按钮：「上传并扫描」full-width 在拖拽区下方

| 状态 | 用户看到 |
|------|----------|
| LOADING | 拖拽区 spinner +「正在上传…」 |
| ERROR | 红框 + 具体原因（非 docx / 过大） |
| SUCCESS | 自动进入 Step2 |

### 4.5 Step2 — 扫描确认 `DeidStepConfirm.vue`

**布局：左右分栏（≥1024px）**

```
┌─ 实体列表 (60%) ─────────┬─ 手动补充 (40%) ────┐
│ 搜索框 + 筛选 chip      │  手动补充（固定标题）  │
│ ┌─ 表格 ─────────────┐ │  规范名称 input       │
│ │☑ 实体 来源 命中 占位│ │  别名 textarea       │
│ │ ...                │ │  ☑ 保存到词库        │
│ └────────────────────┘ │  [ 添加并纳入 ]       │
│  共 N 实体 · M 处命中   │  替换预览（折叠）      │
└────────────────────────┴───────────────────────┘
│ [ 重新扫描 ]              [ 确认并执行脱敏 → ]   │
└──────────────────────────────────────────────────┘
```

**实体表格**：

- 来源列：**Badge** — `词库`（蓝底）/ `手动补充`（橙底）
- 命中 0 的手动项：**警告 icon** + 橙字
- 占位符：`[公司_1]` 用 mono 字体
- 排序：手动补充置顶 → 命中降序（已实现逻辑，需视觉强化）
- 行 hover：浅灰背景；手动行： `--deid-manual-bg`

**替换预览**（折叠 panel）：3 条 before → after 片段，mono 对比

| 状态 | 用户看到 |
|------|----------|
| LOADING | 表格 skeleton 5 行 |
| EMPTY | 「未识别到主体」+ 引导手动补充 |
| SCANNING | 顶栏 progress「扫描中…」 |

### 4.6 Step3 — 导出 `DeidStepExport.vue`

**验证结果卡**（全宽，首要元素）：

```
┌─ 验证通过 ────────────────────────────────┐
│ ✓  已通过残留扫描 · 可安全外发            │
│    替换 156 处 · 12 个实体 · 标准模式     │
└───────────────────────────────────────────┘
```

或失败：

```
┌─ 验证未通过 ──────────────────────────────┐
│ ✕  仍有 3 处疑似未脱敏  [ 展开明细 ]      │
│    ☐ 本人已知晓风险，仍要下载              │
│    备注 [____________]                    │
└───────────────────────────────────────────┘
```

- 主按钮：「下载 ZIP（docx + mapping + 报告）」
- 次按钮：「删除本任务」destructive outline
- **使用须知**：可折叠 `<details>`，默认收起

| 状态 | 用户看到 |
|------|----------|
| RUNNING | 全屏 overlay「正在脱敏…」+ 进度文案 |
| FAILED | 错误卡 + 重试 / 兼容模式说明 |

### 4.7 词库 `DeidLibrary.vue` 重做

**子 Tab**：实体 | 最近加入 | 识别规则 | 白名单

**实体页**：

- 顶：搜索 + pack 下拉 + 「+ 新增实体」
- 列表：**卡片行**（非 bare `<ul>`）— 左规范名，右命中次数 pill，下方 alias chips
- 点击行 → **抽屉**（右侧 400px）编辑别名 textarea、启停开关

**最近加入**：真实数据列表（接 API），空状态「暂无沉淀，处理文档时勾选保存到词库」

**规则/白名单**：表格 + **「测试 regex」面板**（本次重设计一并实现）

**Regex 测试面板**（`DeidLibraryRules.vue` 底部固定区）：

```
┌─ 测试规则 ─────────────────────────────────────┐
│ 选择规则 [下拉 ▼]  或  自定义 regex [________]   │
│ 测试文本 [ textarea 3 行 ]                      │
│ [ 运行测试 ]                                    │
│ 结果：匹配 2 处 — 「华东区域公司」「华北区域公司」│
└─────────────────────────────────────────────────┘
```

- 接现有 API（若有 `POST /api/deid/rules/test`）；若无则 eng 补最小端点
- 结果高亮匹配片段，mono 字体

## 6. 用户旅程

| 步骤 | 用户行为 | 应感受 | 设计元素 |
|------|----------|--------|----------|
| 1 | 打开 /deid | 专业、本地、可信 | 顶栏本地徽章、冷色工作台 |

| 4 | 补漏 | 被支持 | 橙色手动区、保存词库默认开 |

| 6 | 看验证 | 明确能否外发 | 大验证卡，非一行字 |
| 7 | 下载 | 完成感 | 成功态 + 任务侧栏变绿 |
| 8 | 下次再来 | 越来越省事 | 词库命中↑、手动↓ |

## 14. 空状态文案（Pass 2 补充）

| 场景 | 标题 | 说明 | 主操作 |
|------|------|------|--------|
| 无任务 | 还没有脱敏任务 | 上传 Word 审计底稿，系统会扫描可识别主体 | 上传文档 |
| 扫描无命中 | 未发现预设主体 | 可手动补充公司/人名，或检查是否选对了词库包 | 添加实体 |
| 词库空 | 词库还是空的 | 处理文档时勾选「保存到词库」，或在此手动新增 | 新增实体 |
| 最近加入空 | 暂无新沉淀 | 完成脱敏并保存到词库后，这里会显示最近 30 天新增 | 去脱敏 |
| 加载失败 | 无法连接服务 | 请确认后端已启动，或稍后重试 | 重试 |

**Loading 文案**：上传「正在上传…」/ 扫描「正在扫描文档…」/ 执行「正在替换并打包…」（禁止无文案 spinner）。

## 15. 反 AI 模板清单（Pass 4）

**禁止出现**：
- 亮蓝 `#2563eb` 主按钮（改用 `--deid-primary` 海军蓝）
- 三列 feature icon 卡片
- 「欢迎使用」hero 大标题
- 无来源说明的 generic illustration
- 等宽三步骤 pill 无连线

**必须有辨识度**：
- 顶栏「本地处理 · 数据不出服务器」信任徽章
- 占位符 `[公司_1]` 用 mono 字体 — 财务审校师熟悉此格式
- 手动补充区固定橙色调 — 与词库蓝 badge 形成语义对比
- 任务侧栏 countdown「约 X 小时后自动清理」— 提醒数据生命周期

## 16. 键盘与无障碍（Pass 6 补充）

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 新建任务 | `N` | 侧栏聚焦时 |

- 所有 icon-only 按钮：`aria-label` 必填
